Data + applied ML. Independent project, February to April 2026.

Algorithmic Trading Analysis System

I built a Python system that screens the market, records every ticker-date setup it sees, and only keeps a labeled example when a strict, trigger-aware filter says the outcome is unambiguous. Scale was never the point. The filter is.

Role
Independent developer
Stack
Python, Pine Script, SQLite, LightGBM
State
Completed analysis system

System scale, from screened stocks to labeled examples

4,622 stocks screened. The starting universe.
662,678 unique ticker-date observations recorded.
Trigger-aware filter
466,768 unambiguous labeled examples. Roughly seven in ten observations survive.

Of 4,622 stocks screened, the system recorded 662,678 unique ticker-date observations. A trigger-aware filter narrowed that pool down to 466,768 unambiguous labeled examples.

Signal

Multi-factor technical screening flags a breakout or setup candidate across the stock universe.

Outcome

The forward result is labeled and cached in SQLite, every category kept, not just the wins:

  • Win
  • Loss
  • Neutral
Score

LightGBM with time-ordered validation, SHAP analysis, and isotonic calibration turns the label into a probability, then Pine Script backtests it on TradingView.
